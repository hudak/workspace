# Dev Environment Docker Container - Implementation Plan

## Files to Create/Modify

### 1. `Dockerfile` (modify existing)
- Fix `useradd` command: replace `-g developer` with `--user-group` to auto-create the primary group
- Add `AUTHORIZED_KEYS` build arg support for injecting SSH public keys at build time
- Add `COPY` of SSH authorized_keys if provided
- Ensure proper permissions on `.ssh/authorized_keys`

### 2. `docker-compose.yml` (new)
- Build from local Dockerfile
- Mount `/var/run/docker.sock:/var/run/docker.sock` for Docker-in-Docker access
- Mount local dev directory to `/home/developer/workspace`
- Map host port 2222 to container port 22
- Set restart policy
- Allow overriding image name via env/compose override

### 3. `.github/workflows/ci.yml` (new)
- **Trigger**: on push to `main` branch
- **Permissions**: `packages: write`, `contents: read`
- **Steps**:
  1. Checkout code
  2. Docker login to `ghcr.io` using `GITHUB_TOKEN`
  3. Docker metadata action for tags/labels (sha, latest)
  4. Docker build and push to `ghcr.io/<owner>/<repo>:<tag>`
- Use `docker/build-push-action` for efficient caching

### 4. `README.md` (new)
- Project description
- Quick start with docker-compose
- SSH setup instructions (how to add authorized keys)
- GitHub Actions usage
- Local run command without compose

## Implementation Order

1. Fix Dockerfile

3. Create `.github/workflows/ci.yml`
4. Create README.md

## Key Details

### Dockerfile Fixes
```
# Before (broken):
useradd -m -s /usr/bin/zsh -g developer -G docker,sudo developer

# After (fixed):
useradd -m -s /usr/bin/zsh --user-group -G docker,sudo developer
```

### SSH Auth Strategy
- Add `ARG SSH_PUBLIC_KEY` build arg
- If provided, write it to `~/.ssh/authorized_keys` with correct permissions
- Alternative: mount authorized_keys via docker-compose at runtime (documented in README)

### docker-compose.yml Volumes
```yaml
volumes:
  - /var/run/docker.sock:/var/run/docker.sock
  - .:/home/developer/workspace  # local project directory
  # Optional: - ./authorized_keys:/home/developer/.ssh/authorized_keys:ro
```

### GitHub Actions Image Name
- `ghcr.io/${{ github.repository }}:sha-${{ github.sha }}`
- `ghcr.io/${{ github.repository }}:latest`
